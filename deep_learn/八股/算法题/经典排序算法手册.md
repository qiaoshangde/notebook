# 经典排序算法手册

这份笔记用于面试和刷题复习。每个排序算法都按同一套问题理解：

- 它在做什么？
- 核心数据结构或指针是什么？
- 是否稳定？
- 是否原地？
- 时间复杂度和空间复杂度是多少？
- Python 代码怎么写？
- 什么场景适合用？

## 六种已讲排序一句话记忆

- 快速排序：选一个基准值，把小的放左边、大的放右边，基准归位后递归排左右。
- 归并排序：不断把数组二分拆小，再把两个有序数组一层层合并回来。
- 堆排序：先建大根堆，让最大值在堆顶，再不断把堆顶换到末尾并下沉调整。
- 插入排序：左边保持有序，右边每次拿一个数，插入到左边合适位置。
- 冒泡排序：相邻两个数逆序就交换，每一轮把当前最大值冒到最右边。
- 选择排序：每一轮从未排序区间选出最小值，交换到当前最前面。

## 总览表

| 算法       | 平均时间        | 最坏时间        | 额外空间     | 稳定性     | 是否原地 | 记忆关键词                      |
| -------- | ----------- | ----------- | -------- | ------- | ---- | -------------------------- |
| **快速排序** | O(n log n)  | O(n^2)      | O(log n) | 不稳定     | 是    | 最常考分区排序                    |
| **归并排序** | O(n log n)  | O(n log n)  | O(n)     | 稳定      | 否    | 分治，左右有序后合并                 |
| **堆排序**  | O(n log n)  | O(n log n)  | O(1)     | 不稳定     | 是    | 建大根堆，堆顶换到末尾                |
| **插入排序** | O(n^2)      | O(n^2)      | O(1)     | 稳定      | 是    | 像打牌，插入到有序区                 |
| **冒泡排序** | O(n^2)      | O(n^2)      | O(1)     | 稳定      | 是    | 相邻交换，大的冒到后面                |
| **选择排序** | O(n^2)      | O(n^2)      | O(1)     | 不稳定     | 是    | 每轮选最小值放前面                  |
| 计数排序     | O(n + k)    | O(n + k)    | O(k)     | 可稳定     | 否    | 值域小，按数量还原                  |
| 基数排序     | O(d(n + k)) | O(d(n + k)) | O(n + k) | 稳定      | 否    | 按个位、十位、百位排序                |
| 桶排序      | 平均 O(n + k) | O(n^2)      | O(n + k) | 取决于桶内排序 | 否    | 数据均匀分桶                     |
| 希尔排序     | 取决于 gap     | 取决于 gap     | O(1)     | 不稳定     | 是    | 分组插入排序                     |
| 三路快排     | O(n log n)  | O(n^2)      | O(log n) | 不稳定     | 是    | 小于、等于、大于三段                 |
| Timsort  | O(n log n)  | O(n log n)  | O(n)     | 稳定      | 否    | Python 内置 sorted/list.sort |

排序顺序说明：本文按“面试与教材出现频率 + 工程实际使用度”综合排序。快排、归并、堆排最常被系统追问；插入、冒泡、选择是基础必会；计数、基数、桶是典型非比较排序；希尔、三路快排、Timsort 作为补充理解。

## 怎么判断用哪种排序

- 数据量小：插入排序、选择排序、冒泡排序都能写，但插入排序更常用。
- 要稳定：归并排序、插入排序、计数排序稳定版、基数排序、Timsort。
- 要原地且平均快：快速排序。
- 要最坏 O(n log n) 且原地：堆排序。
- 值域很小：计数排序。
- 整数位数有限：基数排序。
- 实际工程 Python：优先 `list.sort()` 或 `sorted()`，它们用 Timsort；面试手写排序时优先练快排、归并、堆排。

## 1. 快速排序

### 思路

快速排序也用分治，但它不是先分半，而是先选一个基准值 `pivot`，把数组分成：

- 小于 pivot 的部分
- pivot 本身
- 大于 pivot 的部分

然后递归排序左右两边。

### Python 代码：原地双指针分区

```python
def quick_sort(nums):
    arr = nums[:]

    def partition(l, r):
        pivot = arr[r]
        i = l
        for j in range(l, r):
            if arr[j] <= pivot:
                arr[i], arr[j] = arr[j], arr[i]
                i += 1
        arr[i], arr[r] = arr[r], arr[i]
        return i

    def sort(l, r):
        if l >= r:
            return
        p = partition(l, r)
        sort(l, p - 1)
        sort(p + 1, r)

    sort(0, len(arr) - 1)
    return arr
```

### 代码执行流程

1. 快排这里使用闭区间 `[l, r]`，也就是处理 `arr[l]` 到 `arr[r]`，左右端点都包含。
2. `partition(l, r)` 选择 `arr[r]` 作为基准。
3. `i` 表示“小于等于 pivot 区间”的下一个位置。
4. `j` 从左到右扫描 `range(l, r)`，也就是扫描到 pivot 前一个位置，不扫描最后的 pivot。
5. 扫描结束后，把 pivot 放到中间位置 `i`。
6. pivot 左边都不大于它，右边都大于它，再递归处理左右两边：`sort(l, p - 1)` 和 `sort(p + 1, r)`。

记忆方法：

- 快排常用闭区间 `[l, r]`：因为它经常直接拿最右端 `arr[r]` 当 pivot。
- 闭区间的空或一个元素：`l >= r`。
- 整个数组：`sort(0, len(arr) - 1)`。
- pivot 已归位后，递归时要跳过它：左边 `sort(l, p - 1)`，右边 `sort(p + 1, r)`。

和归并排序不同：归并更常用左闭右开 `[l, r)`，因为它要把区间切成 `arr[l:mid]` 和 `arr[mid:r]`，正好贴合 Python 切片。

### 坑点

- 快排平均 O(n log n)，最坏 O(n^2)。
- 如果每次 pivot 都选到最小或最大，就会退化。
- 工程中通常随机选 pivot，或者三数取中。
- 快排不稳定，因为分区时会交换远距离元素。

### 为什么快排常写闭区间，归并常写左闭右开

两种区间都正确，核心是“一份代码里前后一致”。常见写法不同，是因为它们的核心动作不同。

快排常写闭区间 `[l, r]`：

- 快排经常直接选最右边元素 `arr[r]` 当 pivot。
- `r` 是真实下标，拿来交换很直观。
- 初始调用是 `sort(0, len(arr) - 1)`。
- 递归出口是 `if l >= r: return`。
- pivot 归位后，左右递归是 `sort(l, p - 1)` 和 `sort(p + 1, r)`。

归并常写左闭右开 `[l, r)`：

- 归并需要把区间从中间切成两段。
- Python 切片天然就是左闭右开：`arr[l:mid]` 和 `arr[mid:r]`。
- 初始调用是 `sort(0, len(arr))`。
- 区间长度直接是 `r - l`。
- 递归出口是 `if r - l <= 1: return`。

记忆卡片：

```text
快排：拿最右下标当 pivot，所以常用闭区间 [l, r]。
归并：像 Python 切片一样切两半，所以常用左闭右开 [l, r)。
```

### 随机 pivot 优化版


## 2. 归并排序

### 思路

归并排序使用分治：

1. 把数组分成左右两半。
2. 分别排序左右两半。
3. 把两个有序数组合并成一个有序数组。

例子：`[5, 1, 4, 2]`

- 分成 `[5,1]` 和 `[4,2]`。
- 排成 `[1,5]` 和 `[2,4]`。
- 合并得到 `[1,2,4,5]`。

### Python 代码

```python
def merge_sort(nums):
    arr = nums[:]

    def sort(l, r):
        if r - l <= 1:
            return arr[l:r]
        mid = (l + r) // 2
        left = sort(l, mid)
        right = sort(mid, r)
        return merge(left, right)

    def merge(left, right):
        ans = []
        i = j = 0
        while i < len(left) and j < len(right):
            if left[i] <= right[j]:
                ans.append(left[i])
                i += 1
            else:
                ans.append(right[j])
                j += 1
        ans.extend(left[i:])
        ans.extend(right[j:])
        return ans

    return sort(0, len(arr))
```

### 代码执行流程

1. `sort(l, r)` 表示排序区间 `arr[l:r]`。
2. 如果区间长度小于等于 1，天然有序，直接返回。
3. 否则取中点 `mid`，递归排序左半和右半。
4. `merge(left, right)` 用两个指针合并两个有序数组。
5. 谁小就先放谁；如果相等，先放左边，所以稳定。

### 坑点

- 归并排序稳定，前提是合并时相等元素优先取左边。
- 额外空间是 O(n)。
- 链表排序常用归并，因为链表不方便随机访问，快排优势不明显。

## 3. 堆排序

### 思路

堆排序用大根堆。大根堆的特点是：父节点总是大于等于子节点，所以堆顶是最大值。

排序过程：

1. 把数组建成大根堆。
2. 把堆顶最大值和数组末尾交换。
3. 缩小堆范围，对堆顶重新下沉。
4. 重复直到排序完成。

### Python 代码：手写大根堆

```python
def heap_sort(nums):
    arr = nums[:]
    n = len(arr)

    def sift_down(i, size):
        while True:
            largest = i
            left = 2 * i + 1
            right = 2 * i + 2
            if left < size and arr[left] > arr[largest]:
                largest = left
            if right < size and arr[right] > arr[largest]:
                largest = right
            if largest == i:
                break
            arr[i], arr[largest] = arr[largest], arr[i]
            i = largest

    for i in range(n // 2 - 1, -1, -1):
        sift_down(i, n)

    for end in range(n - 1, 0, -1):
        arr[0], arr[end] = arr[end], arr[0]
        sift_down(0, end)

    return arr
```

### 代码执行流程

1. `sift_down(i, size)` 表示从位置 `i` 开始向下调整堆。
2. `left = 2 * i + 1`，`right = 2 * i + 2` 是数组表示堆时的左右孩子。
3. 每次找到父节点、左孩子、右孩子里的最大值。
4. 如果最大值不是父节点，就交换并继续向下。
5. 先从最后一个非叶子节点开始建堆。
6. 每次把堆顶最大值换到末尾，再调整剩余堆。

### 坑点

- 堆排序不稳定。
- 建堆是 O(n)，不是 O(n log n)。
- 排序阶段执行 n 次下沉，总体 O(n log n)。

## 4. 插入排序

### 思路

插入排序像整理扑克牌。左边维护一个有序区，每次把右边第一个数插入到左边合适位置。

例子：`[5, 1, 4, 2]`

- 先认为 `[5]` 有序。
- 把 1 插到 5 前面，得到 `[1,5]`。
- 把 4 插到 1 和 5 中间，得到 `[1,4,5]`。
- 把 2 插到 1 和 4 中间。

### Python 代码

```python
def insertion_sort(nums):
    arr = nums[:]
    for i in range(1, len(arr)):
        x = arr[i]
        j = i - 1
        while j >= 0 and arr[j] > x:
            arr[j + 1] = arr[j]
            j -= 1
        arr[j + 1] = x
    return arr
```

### 代码执行流程

1. `i` 指向当前要插入的牌。
2. `x = arr[i]` 暂存当前值，因为后面移动元素会覆盖原位置。
3. `j` 从有序区最后一个位置往前看。
4. 只要 `arr[j] > x`，就把 `arr[j]` 往右挪一格。
5. 找到合适位置后，把 `x` 放到 `arr[j + 1]`。

### 坑点

- 插入排序稳定，因为只有 `arr[j] > x` 才移动；相等时不移动。
- 接近有序时很快，最好时间是 O(n)。
- 小数组常用插入排序作为快速排序、归并排序的优化。

## 5. 冒泡排序

### 思路

冒泡排序每一轮比较相邻两个元素，如果前一个比后一个大，就交换。这样一轮结束后，当前未排序区间里最大的数会被“冒泡”到最右边。

例子：`[5, 1, 4, 2]`

- 第一轮：5 和 1 交换，5 和 4 交换，5 和 2 交换，最大值 5 到最后。
- 第二轮：继续处理前 3 个数。
- 每一轮都会确定一个最大值的位置。

### Python 代码

```python
def bubble_sort(nums):
    arr = nums[:]
    n = len(arr)
    for i in range(n - 1):
        swapped = False
        for j in range(n - 1 - i):
            if arr[j] > arr[j + 1]:
                arr[j], arr[j + 1] = arr[j + 1], arr[j]
                swapped = True
        if not swapped:
            break
    return arr
```

### 代码执行流程

1. `arr = nums[:]` 复制数组，避免修改原数组。
2. 外层循环 `i` 表示已经有多少个最大值放到了末尾。
3. 内层循环只遍历还没排好的部分，所以范围是 `n - 1 - i`。
4. 如果 `arr[j] > arr[j + 1]`，说明相邻两个数逆序，需要交换。
5. 如果一整轮没有交换，说明数组已经有序，可以提前结束。

### 坑点

- 冒泡排序是稳定的，因为相等元素不会交换。
- `n - 1 - i` 不要写错，否则会比较到已经排好的尾部。
- 加 `swapped` 可以优化接近有序数组。

## 6. 选择排序

### 思路

选择排序每一轮从未排序区间里找到最小值，把它交换到当前起点位置。

例子：`[5, 1, 4, 2]`

- 第一轮找最小值 1，放到第 0 位。
- 第二轮在 `[5,4,2]` 里找最小值 2，放到第 1 位。
- 后面继续。

### Python 代码

```python
def selection_sort(nums):
    arr = nums[:]
    n = len(arr)
    for i in range(n - 1):
        min_idx = i
        for j in range(i + 1, n):
            if arr[j] < arr[min_idx]:
                min_idx = j
        arr[i], arr[min_idx] = arr[min_idx], arr[i]
    return arr
```

### 代码执行流程

1. `i` 表示当前要填的位置。
2. `min_idx` 记录未排序区间里最小元素的下标。
3. 内层循环从 `i + 1` 往后找更小的数。
4. 找完后，把 `arr[min_idx]` 和 `arr[i]` 交换。
5. 每轮结束后，`arr[:i+1]` 都是排好序的。

### 坑点

- 选择排序通常不稳定。比如 `[2a, 2b, 1]`，第一轮会把 `1` 和 `2a` 交换，两个 2 的相对顺序可能改变。
- 它的交换次数少，但比较次数固定是 O(n^2)。

## 7. 计数排序

### 思路

计数排序适合整数且值域不大的数据。它不比较元素大小，而是统计每个值出现了多少次。

例子：`[2, 5, 3, 0, 2, 3, 0, 3]`

- 0 出现 2 次。
- 2 出现 2 次。
- 3 出现 3 次。
- 5 出现 1 次。
- 按值从小到大还原即可。

### Python 代码：简单版

```python
def counting_sort(nums):
    if not nums:
        return []
    min_val = min(nums)
    max_val = max(nums)
    count = [0] * (max_val - min_val + 1)
    for x in nums:
        count[x - min_val] += 1

    ans = []
    for i, c in enumerate(count):
        ans.extend([i + min_val] * c)
    return ans
```

### Python 代码：稳定版

稳定版适合排序对象，比如按年龄排序学生，并保持同年龄学生的原相对顺序。

```python
def counting_sort_stable(nums):
    if not nums:
        return []
    min_val = min(nums)
    max_val = max(nums)
    count = [0] * (max_val - min_val + 1)
    for x in nums:
        count[x - min_val] += 1

    for i in range(1, len(count)):
        count[i] += count[i - 1]

    ans = [0] * len(nums)
    for x in reversed(nums):
        idx = x - min_val
        count[idx] -= 1
        ans[count[idx]] = x
    return ans
```

### 代码执行流程

1. 先找最小值和最大值，确定计数数组长度。
2. `count[x - min_val]` 记录值 `x` 出现了多少次。
3. 简单版直接按次数还原。
4. 稳定版先做前缀和，让 `count[i]` 表示小于等于当前值的元素总数。
5. 从右往左放回答案数组，可以保持稳定性。

### 坑点

- 值域大时不适合，比如只有 10 个数但范围是 0 到 10^9。
- 计数排序只能直接处理离散整数；对象排序需要把 key 映射成整数。

## 8. 基数排序

### 思路

基数排序适合非负整数。它按位排序：

1. 先按个位排序。
2. 再按十位排序。
3. 再按百位排序。
4. 直到最高位。

每一位排序必须稳定，否则低位已经排好的顺序会被破坏。

### Python 代码：非负整数

```python
def radix_sort(nums):
    if not nums:
        return []
    arr = nums[:]
    max_val = max(arr)
    exp = 1
    while max_val // exp > 0:
        arr = counting_sort_by_digit(arr, exp)
        exp *= 10
    return arr


def counting_sort_by_digit(nums, exp):
    count = [0] * 10
    for x in nums:
        digit = (x // exp) % 10
        count[digit] += 1

    for i in range(1, 10):
        count[i] += count[i - 1]

    ans = [0] * len(nums)
    for x in reversed(nums):
        digit = (x // exp) % 10
        count[digit] -= 1
        ans[count[digit]] = x
    return ans
```

### 代码执行流程

1. `exp = 1` 表示先看个位。
2. `(x // exp) % 10` 取出当前位数字。
3. 对当前位做稳定计数排序。
4. `exp *= 10`，下一轮看十位、百位。
5. 所有位处理完后，数组整体有序。

### 坑点

- 基数排序依赖稳定排序。
- 上面代码只处理非负整数。
- 如果有负数，可以把负数和非负数分开处理，负数按绝对值排序后反转。

## 9. 桶排序

### 思路

桶排序把数据分到多个桶里，每个桶内部排序，最后按桶顺序拼接。

它适合数据分布比较均匀的场景。比如一堆 0 到 1 之间的小数，可以按区间分桶。

### Python 代码

```python
def bucket_sort(nums, bucket_size=5):
    if not nums:
        return []

    min_val = min(nums)
    max_val = max(nums)
    bucket_count = (max_val - min_val) // bucket_size + 1
    buckets = [[] for _ in range(bucket_count)]

    for x in nums:
        idx = (x - min_val) // bucket_size
        buckets[idx].append(x)

    ans = []
    for bucket in buckets:
        ans.extend(insertion_sort(bucket))
    return ans
```

### 代码执行流程

1. 先用 `min_val` 和 `max_val` 确定数据范围。
2. 按 `bucket_size` 计算需要多少个桶。
3. 每个数通过 `(x - min_val) // bucket_size` 找到自己的桶。
4. 每个桶内部单独排序。
5. 从小桶到大桶依次拼接，就是整体有序结果。

### 坑点

- 桶排序依赖数据分布。如果所有数都落进同一个桶，就退化成桶内排序。
- 桶内可以用插入排序，也可以用 Python 内置排序。
- `bucket_size` 会影响性能。

## 10. 希尔排序

### 思路

希尔排序是插入排序的升级版。它先让相隔 `gap` 的元素组成一组，对每组做插入排序；然后逐渐缩小 `gap`，最后当 `gap = 1` 时，整个数组已经接近有序，插入排序会更快。

例子：`[8, 9, 1, 7, 2, 3, 5, 4, 6, 0]`

- 先用较大 gap 粗略调整远距离逆序。
- 再用较小 gap 细调。
- 最后 gap 为 1，完成普通插入排序。

### Python 代码

```python
def shell_sort(nums):
    arr = nums[:]
    n = len(arr)
    gap = n // 2
    while gap > 0:
        for i in range(gap, n):
            x = arr[i]
            j = i
            while j >= gap and arr[j - gap] > x:
                arr[j] = arr[j - gap]
                j -= gap
            arr[j] = x
        gap //= 2
    return arr
```

### 代码执行流程

1. `gap = n // 2`，先让距离较远的元素互相比较。
2. `for i in range(gap, n)` 遍历每个需要插入的元素。
3. `j -= gap` 表示不是和前一个元素比，而是和同组前一个元素比。
4. 每轮 gap 缩小一半。
5. 最后一轮 `gap = 1`，就是普通插入排序。

### 坑点

- 希尔排序不稳定，因为远距离交换可能改变相等元素顺序。
- 复杂度和 gap 序列有关，面试中通常记“优于 O(n^2)，但不是稳定 O(n log n)”。

## 11. 三路快速排序

### 思路

普通快排在重复元素很多时效率可能变差。三路快排把数组分成三段：

- `< pivot`
- `== pivot`
- `> pivot`

这样等于 pivot 的大量元素不用继续递归。

### Python 代码

```python
def quick_sort_3way(nums):
    arr = nums[:]

    def sort(l, r):
        if l >= r:
            return
        pivot = arr[l]
        lt = l
        i = l + 1
        gt = r
        while i <= gt:
            if arr[i] < pivot:
                arr[lt], arr[i] = arr[i], arr[lt]
                lt += 1
                i += 1
            elif arr[i] > pivot:
                arr[i], arr[gt] = arr[gt], arr[i]
                gt -= 1
            else:
                i += 1
        sort(l, lt - 1)
        sort(gt + 1, r)

    sort(0, len(arr) - 1)
    return arr
```

### 代码执行流程

1. 三路快排也使用闭区间 `[l, r]`。
2. `lt` 左边都是小于 pivot 的元素。
3. `gt` 右边都是大于 pivot 的元素，初始 `gt = r`。
4. `i` 扫描中间未知区域。
5. 如果 `arr[i] < pivot`，交换到左边，`lt` 和 `i` 都右移。
6. 如果 `arr[i] > pivot`，交换到右边，只移动 `gt`，因为换过来的元素还没检查。
7. 如果相等，只移动 `i`。

### 坑点

- `arr[i] > pivot` 时不能立刻 `i += 1`，因为从右边换来的元素还没判断。
- 重复元素多时，三路快排非常好用。

## 12. Timsort：Python 内置排序

### 思路

Python 的 `list.sort()` 和 `sorted()` 使用 Timsort。它是归并排序和插入排序的混合优化，特别擅长处理真实世界中“部分有序”的数据。

### Python 用法

```python
nums = [5, 1, 4, 2]

sorted_nums = sorted(nums)
nums.sort()
```

### 按 key 排序

```python
students = [
    {"name": "alice", "age": 18},
    {"name": "bob", "age": 17},
    {"name": "cindy", "age": 18},
]

students_by_age = sorted(students, key=lambda x: x["age"])
```

### 多关键字排序

```python
items = [("apple", 3), ("banana", 2), ("apple", 1)]
ans = sorted(items, key=lambda x: (x[0], x[1]))
```

### 代码执行流程

1. `sorted(nums)` 返回新列表，不修改原数组。
2. `nums.sort()` 原地排序，返回 `None`。
3. `key` 会先把每个元素映射成用于比较的值。
4. Python 排序是稳定的；key 相等时，原相对顺序不变。
5. 多关键字排序可以用元组，先比第一个，再比第二个。

### 坑点

- `nums = nums.sort()` 是错误写法，因为 `list.sort()` 返回 `None`。
- 降序用 `reverse=True`。
- 自定义对象排序优先用 `key`，少用比较函数。

## 面试常问对比

### 快排 vs 归并

| 对比点 | 快速排序 | 归并排序 |
| --- | --- | --- |
| 平均时间 | O(n log n) | O(n log n) |
| 最坏时间 | O(n^2) | O(n log n) |
| 空间 | O(log n) 递归栈 | O(n) 辅助数组 |
| 稳定性 | 不稳定 | 稳定 |
| 适合数组 | 很适合 | 也适合 |
| 适合链表 | 一般 | 很适合 |

记忆：快排靠分区，空间省但最坏会退化；归并靠合并，稳定但需要额外空间。

### 堆排 vs 快排

| 对比点 | 堆排序 | 快速排序 |
| --- | --- | --- |
| 平均时间 | O(n log n) | O(n log n) |
| 最坏时间 | O(n log n) | O(n^2) |
| 空间 | O(1) | O(log n) |
| 稳定性 | 不稳定 | 不稳定 |
| 实际速度 | 通常慢于快排 | 缓存友好，通常更快 |

记忆：堆排最坏稳，快排平均快。

### 插入排序为什么有用

插入排序虽然最坏 O(n^2)，但它在小数组和接近有序数组上很快。很多工程排序会在数组规模很小时切换到插入排序。

## 刷题模板记忆

### 比较排序模板

适用：冒泡、选择、插入、希尔、归并、快排、堆排。

核心问题：元素之间通过比较大小决定顺序。

```python
if a > b:
    # 调整顺序
    pass
```

### 分治排序模板

适用：归并排序、快速排序。

```python
def sort(l, r):
    if l >= r:
        return
    mid_or_pivot = 分割点
    sort(l, mid_or_pivot)
    sort(mid_or_pivot + 1, r)
```

区别：

- 归并排序：先递归排左右，再合并。
- 快速排序：先分区，再递归排左右。

### 非比较排序模板

适用：计数排序、桶排序、基数排序。

核心问题：利用数字本身的范围、桶编号、位数，而不是两两比较。

```python
count = [0] * k
for x in nums:
    count[index(x)] += 1
```

## 记忆卡片

- 冒泡：相邻逆序就交换，每轮最大到尾部。
- 选择：每轮选最小，放到当前起点。
- 插入：左边有序，右边拿一张牌插进去。
- 希尔：隔 gap 做插入排序，gap 慢慢变小。
- 归并：分到最小，再两个有序数组合并。
- 快排：选 pivot，左小右大，再递归。
- 三路快排：重复多时分成小于、等于、大于。
- 堆排：大根堆顶是最大值，换到末尾。
- 计数：值域小，数每个值出现几次。
- 桶排：分桶，桶内排序，再拼接。
- 基数：个位、十位、百位逐位稳定排序。
- Timsort：Python 内置稳定排序，真实工程优先用。

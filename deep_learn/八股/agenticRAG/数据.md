**seed QA 的清洗**
一开始，每个 chunk 会交给 LLM，让它最多生成 3 条原子 QA。这里 prompt 已经先做了一层软约束，要求每个 QA 只对应一个不可拆分的事实，答案必须来自文档，最好是数字、日期或唯一实体名，不能是主观判断，也不能是模糊表达。比如它希望生成的是“永辉超市 2025 年上半年经营活动现金流量净额是多少”，而不是“这家公司经营情况怎么样”。

LLM 生成之后，代码会再做硬规则过滤。第一，答案不能太长，代码里是 `len(_tokens(answer)) >= 10` 就丢弃，也就是答案必须足够简洁。第二，答案不能直接出现在问题里，否则就是答案泄露。比如问题里已经写了“经营活动现金流量净额为 1234567890 元的是哪家公司”，这种就不合格。第三，答案里的 token 不能和问题重叠过多，如果答案一半以上 token 都出现在问题里，也认为有泄露风险。第四，答案不能包含 `and/or/&` 这类连接词，避免一个 seed QA 里包含多个事实。因为 seed QA 要作为后续多跳合成的基本单元，必须是原子事实。第五，问题不能引用“document / article / according”这类词，避免问题变成“根据本文档……”这种不自然的查文档问题。第六，会过滤掉 “full name / original name / alternate name” 这类要求全名、别名的问题，因为这类问题容易模糊或变成纯文本表述匹配。

同时，在同一个 chunk 内还会做去重。代码会维护 `seen_answers` 和 `seen_questions`，如果归一化后的答案或问题已经出现过，就丢弃，避免一个 chunk 生成多条重复 QA。

通过这些规则后，还会做答案精炼。也就是把 LLM 生成的原始答案再交给 refine prompt，压缩成最直接的答案。例如原始答案可能是“2025 年上半年公司实现营业收入 35.8 亿元，同比增长 12.5%”，如果问题问的是同比增长多少，精炼后就只保留“12.5%”。精炼时也会统一一些格式，比如百分比、日期、单位。精炼后还会再检查一次答案长度，如果仍然超过 10 tokens，就继续丢弃。

先用 prompt 约束 LLM 生成原子、可验证的单跳 QA，再用代码过滤掉答案过长、答案泄露、问题答案高度重叠、复合答案、文档引用式问题和重复 QA，最后通过答案精炼把答案压缩成标准短答案。这样得到的 seed QA 才能作为后续多跳合成的可靠基础。‘

**多跳qa清洗**
一条多跳 QA 合成出来后，我不会直接放进训练集，而是先做多层质量过滤。首先是四重验证：第一步检查语义逻辑是否成立，比如 comparison 问题必须比较同一类财务指标，inference 问题前后 hop 之间必须有真实依赖，不能是强行拼接；第二步不给任何文档，直接让 LLM 回答，如果它不看 evidence 也能答对，说明这题不需要 RAG，可能是常识题或答案泄露题，就会删除；第三步做部分文档检查，假设这条 QA 需要 N 个 gold chunks，我会每次只给 N-1 个文档，也就是故意拿掉一个证据块，如果少了任意一个证据 LLM 还能答对，说明某些 hop 是冗余的，不是真正多跳；第四步把所有 gold chunks 都给 LLM，如果这时还答不对，说明问题本身可能有歧义、答案标错或证据不足，也会过滤。通过四重验证后，再做规则清洗，比如 3-hop 以上如果最后一跳只是“期末余额是多少”“总计是多少”这种 trivial 查表，就认为可能是在凑 hop；如果整条 QA 所有 hop 都是简单查表，也会删除；同时还会做问题前缀去重和 chunk 重叠去重，如果 final question 前 80 个字符重复，或者和已有样本使用的 gold chunks 重叠超过 80%，也会过滤。规则清洗之后，再用 LLM Judge 从答案正确性、多跳必要性、问题清晰度三个维度打分，总分不达标就删除。最后还会做搜索可达性过滤，也就是对每一跳的 hop question 调用 keyword search、semantic search、graph search 和 hybrid search，看对应的 gold chunk 能不能出现在 top-k 结果里；如果某一跳标准证据完全搜不到，这条 QA 也不会进入训练集。这样最后保留下来的样本，既不是常识题，也不是单文档题，不是简单凑 hop，而且答案能被证据支持，当前检索系统也确实有能力搜到它需要的证据。

面试时可以浓缩成一句主线：

> 我不是直接让模型随便生成 QA，而是先把财报切成 chunk，生成单跳 seed QA，再通过检索扩展成多跳 QA；合成过程中用四重验证保证问题真的需要多跳证据，之后再用规则清洗、LLM Judge 打分和搜索可达性过滤层层筛选，最终得到 797 条训练样本和 185 条测试样本。

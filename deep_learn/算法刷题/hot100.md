# 2、字母异位词分组
![[Pasted image 20260131144041.png]]


用到了字典哈希。
先遍历对所有单词进行排序，然后以排序后的词作为键，值是一个列表，然后判断字典中是否有这个键，然后将其添加到这个排序后的键对应的值（列表）中。这样所有的异位词组都会存到这个键对应的列表中。只需要取值，然后进行list就可以了。

代码
```python
class Solution:

    def groupAnagrams(self, strs: List[str]) -> List[List[str]]:
        mp = collections.defaultdict(list)
        for st in strs:

            key = "".join(sorted(st))

            mp[key].append(st)

        return list(mp.values())
```

在代码中用到了一个叫collections的库，这个库可以帮助我少点检查是否有这个键的操作。直接向里面添加，如果有就可以对这个键对应的值进行操作，如果没有，会新建一个这个键，直接进行操作。

defaultdict（list）这个括号中填写一个数据类型。一旦检测到没有这个键，新建的这个键对应的值的数据类型就是这个默认的数据类型。
soted 就是进行排序。不管扔给他的是字符串还是列表，其返回都是一个列表。




最快的排序的复杂度是O（nlogn）
hashset的查找速度是O（1）。直接执行以此查询就行。


什么是hashset。就是一个只有键，没有值得哈希表。（可以用来执行自动去重）
并且无序无索引。就像一个大麻袋，里面有很多数字，你只能问里面有没有这个数字。但是你不能问这个里面第一个的数字是多少。

hashmap字典，hashset是集合。

# 4、移动0
这是一个关于快慢指针的题目。

![[Pasted image 20260201134457.png]]



直觉解法：遇到0就给它移动到最后。（计算复杂度太高，o（n2））

```python
# 你的思路的代码实现（虽然能跑，但效率低）
class Solution:
    def moveZeroes(self, nums: List[int]) -> None:
        i = 0
        loop_count = len(nums)
        
        # 我们得手动控制循环次数，不然死循环
        processed = 0 
        while processed < loop_count:
            if nums[i] == 0:
                # 1. 把 0 拿出来 (这一步很慢！O(N))
                nums.pop(i)
                # 2. 加到最后
                nums.append(0)
                # 3. 注意！因为删了一个元素，后面的补上来了
                # 所以 i 不需要 +1，我们要检查补上来的这个新元素
            else:
                # 只有不是 0 的时候，指针才往后走
                i += 1
            
            processed += 1

```

双指针法

```python
class Solution:

    def moveZeroes(self, nums: List[int]) -> None:

        """

        Do not return anything, modify nums in-place instead.

        """

        # slow 指针：指向下一个存放“非零元素”的位置

        slow = 0

        # fast 指针：负责遍历数组寻找“非零元素”

        for fast in range(len(nums)):

            # 只有当找到非零元素时，才做事

            if nums[fast] != 0:

                # 把找到的这个非零元素，和 slow 位置的元素交换

                # 这样非零的就被挪前面了，0 就被换后面去了

                nums[slow], nums[fast] = nums[fast], nums[slow]

                # 既然这个位置坐好了人，slow 就往后移一位，准备迎接下一个

                slow += 1
```

这是一个关于快慢指针的题目。在这个题中有两个下标，一个下标只负责查看是不是0.如果不是0，就移动到前面（慢指针指向的位置，相当于快慢指针指向的数字交换位置）。如果是0就不动。

复杂度为O（n）。原地进行操作。
也不能移动O到最后，交换位置。这样会导致非0元素的原始顺序会发生变化。


# 5、盛水最多的容器
![[Pasted image 20260201140345.png]]


题目分析，从左右头开始扫描。必须淘汰矮的一边，因为只要矮的呢一边存在。不管另一边移动后下一个是多少，都没有用，因为这个矮的就是限制的最大瓶颈。

```python
class Solution:
    def maxArea(self, height: List[int]) -> int:
        # 1. 左右护法就位
        left = 0
        right = len(height) - 1
        
        max_water = 0
        
        # 2. 只要没撞上，就一直PK
        while left < right:
            # 计算当前宽度
            width = right - left
            
            # 计算当前水位（取决于短板）
            # 这一步通过比较，顺便决定了谁该被淘汰
            if height[left] < height[right]:
                current_height = height[left]
                # 左边矮，记录下当前面积，然后淘汰左边
                max_water = max(max_water, width * current_height)
                left += 1
            else:
                current_height = height[right]
                # 右边矮（或一样高），记录下当前面积，然后淘汰右边
                max_water = max(max_water, width * current_height)
                right -= 1
                
        return max_water
```

两个指针分别指向左右，然后进行移动，只需要对数据扫描一遍，就可以找到最多的盛水容器。



# 6、三数之和


![[Pasted image 20260201142238.png]]




这是一个双指针的题目
```python
class Solution:
    def threeSum(self, nums: List[int]) -> List[List[int]]:
        # 1. 如果还没开始这就少于3个人，直接解散
        if not nums or len(nums) < 3:
            return []
        
        # 2. 关键第一步：排序！
        # 只有排序了，才能用双指针，才能方便去重
        nums.sort()
        
        result = []
        n = len(nums)
        
        # 3. 遍历每一个“老大”
        for i in range(n):
            # 如果老大自己都大于0，那后面肯定都大于0（因为排过序了）
            # 三个大于0的数加起来不可能等于0，直接结束
            if nums[i] > 0:
                break
                
            # 4. 老大去重：如果这个数和上一个数一样，跳过
            # i > 0 是为了保证 i-1 有效
            if i > 0 and nums[i] == nums[i-1]:
                continue
                
            # 5. 定义左右护法（双指针）
            # L 在老大右边第一个，R 在最后
            L = i + 1
            R = n - 1
            
            while L < R:
                total = nums[i] + nums[L] + nums[R]
                
                if total == 0:
                    # 找到了一组！
                    result.append([nums[i], nums[L], nums[R]])
                    
                    # 6. 护法去重（关键细节！）
                    # 比如 [-1, 0, 0, 0, 1]，找到一个0后，后面两个0都要跳过
                    while L < R and nums[L] == nums[L+1]:
                        L += 1
                    while L < R and nums[R] == nums[R-1]:
                        R -= 1
                    
                    # 找到答案后，两人同时收缩，找下一组
                    L += 1
                    R -= 1
                    
                elif total < 0:
                    # 结果太小了，把左边的（小的）变大一点
                    L += 1
                else:
                    # 结果太大了，把右边的（大的）变小一点
                    R -= 1
                    
        return result
```


首先对这个列表进行排序，从小到大排序。
然后进行循环，如果这个数跟左右两个指针指向的数的和为0，那么就是找到了一组。
需要注意的是：左右指针相遇就是两个指针已经对整个列表进行了扫描，已经寻找完毕。
循环时，这个数和上一个数不能相同。相同的话会找到同样的三个数。（去重）
左右指针移动的时候也要去重。

```python
            return 0

        left = 0

        right = len(height) - 1

        # 2. 定义左右两边的“墙”的高度

        left_max = 0

        right_max = 0

        water = 0

        # 3. 往中间逼近

        while left < right:

            # 更新左右两边的最高墙

            left_max = max(left_max, height[left])

            right_max = max(right_max, height[right])

            # 4. 核心逻辑：谁矮算谁

            if left_max < right_max:

                # 左边墙矮，说明左边这个位置的水位由 left_max 决定

                # 累加水量：瓶颈高度 - 柱子本身高度

                water += left_max - height[left]

                left += 1

            else:

                # 右边墙矮，说明右边这个位置的水位由 right_max 决定

                water += right_max - height[right]

                right -= 1

        return water
```

# 8、无重复字符的最长子串
![[Pasted image 20260201164831.png]]

```python

class Solution:
    def lengthOfLongestSubstring(self, s: str) -> int:
        # 1. 也是用 Set 做“记账本”，查重速度快 O(1)
        window_chars = set()
        
        # 2. 定义左指针
        left = 0
        max_len = 0
        
        # 3. 右指针主动去探索 (0 到 结束)
        for right in range(len(s)):
            current_char = s[right]
            
            # 4. 如果发现新来的字符已经在窗口里了（撞车了！）
            # 就要不断缩小左边，直到把那个重复的字符“挤出去”
            while current_char in window_chars:
                # 移除最左边的字符
                window_chars.remove(s[left])
                # 左指针向右缩
                left += 1
            
            # 5. 现在安全了，把新字符加进来
            window_chars.add(current_char)
            
            # 6. 量一下现在的宽度，挑战一下历史纪录
            # 宽度计算公式：right - left + 1
            max_len = max(max_len, right - left + 1)
            
        return max_len
```


在这个题目中，维护了两个指针，两个指针合在一起，相当于是一个滑动的窗口，一开始只移动右窗口，添加新的元素。一旦发现添加的元素中有已经含有的元素。则不断移动左窗口，知道把重复的旧元素排除出去。这个过程中，时刻计算窗口的长度，右指针减去左指针加一，并于已知的最大窗口进行比较，遍历一遍，即可得到最大的窗口。



# 9、找到字符串中的所有字母异位词
![[Pasted image 20260201165945.png]]



这个题目还是滑动窗口的题目。

```python
class Solution:
    def findAnagrams(self, s: str, p: str) -> List[int]:
        n, m = len(s), len(p)
        
        # 1. 边界条件：如果 s 比 p 还短，那肯定没有，直接回空
        if n < m:
            return []
        
        # 2. 准备两个 DNA 记录表 (长度26的数组，初始都是0)
        p_count = [0] * 26
        window_count = [0] * 26
        
        # 3. 先把 p 的 DNA 录入，同时把 s 的前 m 个字符（第一个窗口）录入
        for i in range(m):
            p_count[ord(p[i]) - ord('a')] += 1
            window_count[ord(s[i]) - ord('a')] += 1
            
        res = []
        
        # 4. 检查第一个窗口是否匹配
        if p_count == window_count:
            res.append(0)
            
        # 5. 开始滑动！从第 m 个字符开始，一直滑到最后
        # i 代表当前窗口【新加入】的那个字符的索引
        for i in range(m, n):
            # A. 进货：把新字符加入窗口
            new_char = s[i]
            window_count[ord(new_char) - ord('a')] += 1
            
            # B. 出货：把最左边被抛弃的字符移出窗口
            # 最左边的字符索引是 i - m
            old_char = s[i - m]
            window_count[ord(old_char) - ord('a')] -= 1
            
            # C. 比对：看看现在的窗口和 p 是否一样
            # 在 Python 里，比较两个列表的内容是否相同，直接用 == 就行，效率很高
            if window_count == p_count:
                # 记录起始索引 (就是 i - m + 1)
                res.append(i - m + 1)
                
        return res

```

在这个题目中，就是制作了一个和字串一样大小的窗口。然后制作了一个长为26的列表，然后默认第一位置的值就是a出现的次数。第二个位置的值就是b出现的次数，
 两个列表完全一样的时候，就说明这个符合添加，字串找到。记录开始的索引。 
 
 通过ord（）函数，就可以求出对应字符的ascll码。


# 10、和为K的子数组
![[Pasted image 20260201171413.png]]



看到题的第一反应是使用滑动窗口来做。但是不可以，因为有负的值。

一旦遇到负值，滑动窗口的指针就不知道移动哪一个。


```python
from collections import defaultdict

class Solution:
    def subarraySum(self, nums: List[int], k: int) -> int:
        # 1. 记账本：记录 {前缀和 : 出现的次数}
        # 核心细节：必须预设 {0: 1}
        # 含义是：前缀和为 0 的情况已经出现过 1 次了（就是刚开始还没动的时候）
        prefix_map = defaultdict(int)
        prefix_map[0] = 1
        
        current_sum = 0
        count = 0
        
        # 2. 遍历数组
        for num in nums:
            # 计算当前走到这里的累加和
            current_sum += num
            
            # 3. 核心公式查询：
            # 想知道有没有一个子数组和是 k，
            # 就看以前有没有出现过前缀和等于 (current_sum - k)
            target = current_sum - k
            
            # 如果以前出现过 target，说明中间这一段的和刚好就是 k
            if target in prefix_map:
                count += prefix_map[target]
            
            # 4. 把当前的前缀和记录到账本里
            prefix_map[current_sum] += 1
            
        return count

```


在这个题目中，建立了一个字典，key为前缀和，V为这个前缀和出现了几次。（可以观察到，如果值全都是正值，那么前缀和就是单调的，在这个字典中所有的值对应的都是1.）正是因为有负值的存在，所以导致前缀和的value不全为1.

一直求前缀和，找出当前前缀和减去之前某个位置的前缀和，就是这个子数组的前缀和。
因此，就可以转化为由当前前缀和减去k，就是某个要找的前缀和。如果字典中有，就说明这个前缀和出现过，就说明，字串存在。


# 11、滑动窗口最大值
![[Pasted image 20260202131218.png]]




```python
from collections import deque

class Solution:
    def maxSlidingWindow(self, nums: List[int], k: int) -> List[int]:
        # 这里的 q 就是那个“精英名单”，里面存的是【下标】！
        # 为什么要存下标？因为我们要通过下标算它是不是过期了。
        q = deque() 
        result = []
        
        for i in range(len(nums)):
            # 1. 【踢弱者】
            # 只要队列里有人，而且队尾的那个人比新来的 nums[i] 小
            # 那队尾那个人就永远没机会出头了，直接淘汰
            while q and nums[q[-1]] < nums[i]:
                q.pop() 
            
            # 2. 【进新人】
            # 现在的 nums[i] 有资格排队了
            q.append(i)
            
            # 3. 【清过气】
            # 队头是最老的。如果队头的下标 等于 i - k
            # 说明它刚好滑出了窗口的左边界，必须退休
            # 举例：k=3，当前 i=3 (第4个数)，那 i-k=0 (第0个数) 必须走了
            if q[0] == i - k:
                q.popleft()
            
            # 4. 【记录结果】
            # 从第 k-1 个元素（也就是窗口刚满）开始，每次都记下队头
            if i >= k - 1:
                result.append(nums[q[0]])
                
        return result
```


在这个题目中，考到了滑动窗口的知识点。
一个个加入到窗口。如何加入？，如果这个值比窗口内的所有值都大。就说明窗口内的值不起作用了。就把这个放到队头。其余的全部失效。如果不比第一个值大，但是比其余的值大，在窗口内时，也可以加入，但是也要按照大小顺序。（因为第一个有可能不被挤出但是时效到了，被踢出来了，这个时候，排名第二的数字，就可以顺利继位，变成第一个）
但是这个值当队头是有时效性的。如果在不被后加的值踢出的情况下。其最多可以呆一个滑动窗口的大小。（也就是新来的值的下标减去窗口的大小为最大值得下标的时候，意味着它已经呆了一个滑动窗口的大小了，马上要出窗口了）

进队前铲除异己（比我弱的全踢），进队后听天由命（等着前面的大佬退休）。

知识补充：在进队的时候，会把自己小的值全部踢出。因为这个时候，比自己小的值，再也不会有当队头的机会了，所以会直接踢出。
可以理解为，这个队列 就是一个最大值，以及最大值候选队列。每个一进来，就跟最后一个比，比它小的踢出，逐个和前面的比，直到找到自己的位置（其后面的全部踢出）。然后等待队列前面的大佬退休。自己继位。

```python
            # --- 进货环节 ---

            c = s[right]

            right += 1

            # 如果进的货是欠条里需要的，就记账

            if c in need:

                window[c] += 1

                # 如果这个字符的数量刚好凑够了，valid + 1

                if window[c] == need[c]:

                    valid += 1

            # --- 断舍离环节 ---

            # 当所有债务都还清了 (valid == len(need))，尝试缩小窗口

            while valid == len(need):

                # 1. 更新最小长度

                if right - left < min_len:

                    start = left

                    min_len = right - left

                # 2. 准备扔掉左边的货

                d = s[left]

                left += 1

                # 如果扔掉的货是欠条里的

                if d in need:

                    # 如果扔之前刚好够，扔了就不够了，valid - 1

                    if window[d] == need[d]:

                        valid -= 1

                    window[d] -= 1

        return "" if min_len == float('inf') else s[start : start + min_len]
```


这个题目，仍然是滑动窗口的题目

这个题目维护了两个值，一个是当前窗口内的字母的分布。（每个字母出现的字数）。
一个是（valid 表示：已经有几种字符的数量达标了？，比如t为AAB，这个时候need={‘A’：2，‘B’：1}，如果valid等于2，就表示A有两个B有一个）。

不断地将字母加入到窗口中，并统计符合条件的字母个数。合格时，就记录当前的下表和长度。然后将前面的出窗口。这里分为三种情况。一种是不是需要的字母，直接出。一种是是需要的字母，但是其出去后，窗口内的字母仍满足条件（这个字母本来就有富裕，出去一个没问题）。还有一种是是需要的字母，并且其一出去，条件就不满足，知道新的这个字母进来。

每次找到符合条件的情况，都记录下表和长度，直到找到最短的一版。（一旦新版比旧版短，就更新）。

```python
class Solution:
    def maxSubArray(self, nums: List[int]) -> int:
        # 1. 初始化
        # pre_sum 表示：必须包含当前这个数字的话，前面的最大和是多少
        # 刚开始就是第一个数
        pre_sum = nums[0]
        
        # max_sum 记录历史出现过的巅峰值
        max_sum = nums[0]
        
        # 2. 从第二个数开始遍历
        for i in range(1, len(nums)):
            # --- 核心决策 ---
            # 如果之前的累加和大于0，说明是“良性资产”，我们就加上它
            if pre_sum > 0:
                pre_sum = pre_sum + nums[i]
            # 如果之前的累加和小于等于0，说明是“不良资产”，只会拖累我
            # 那就不带它玩了，从当前数字 nums[i] 另起炉灶
            else:
                pre_sum = nums[i]
            
            # --- 记录巅峰 ---
            # 每次走一步，都要看看现在的身价是不是历史最高的
            max_sum = max(max_sum, pre_sum)
            
        return max_sum
```

```python
class Solution:
    def merge(self, intervals: List[List[int]]) -> List[List[int]]:
        # 1. 没有任何区间，直接返回空
        if not intervals:
            return []
        
        # 2. 关键一步：排序！
        # 按照每个区间的【左端点】从小到大排序
        # x[0] 表示取每个子列表的第0个元素（开始时间）
        intervals.sort(key=lambda x: x[0])
        
        merged = []
        
        # 3. 遍历每一个区间
        for interval in intervals:
            # 如果列表为空，或者当前区间的开始时间 > 上一个区间的结束时间
            # 说明【没有重叠】，直接把当前区间加进去
            if not merged or interval[0] > merged[-1][1]:
                merged.append(interval)
            else:
                # 否则，说明【有重叠】
                # 我们不需要添加新区间，而是修改上一个区间的结束时间
                # 谁结束得晚，就听谁的（取最大值）
                merged[-1][1] = max(merged[-1][1], interval[1])
        
        return merged
```


人的常理，像时间即可。先对这个按开始时间进行排序，然后一个个加入合并后的区间，一旦合并后的区间最后一个和新的区间有重叠就取最大的一个。

```python
class Solution:
    def rotate(self, nums: List[int], k: int) -> None:
        """
        Do not return anything, modify nums in-place instead.
        """
        n = len(nums)
        # 1. 处理 k 比 n 大的情况
        k = k % n
        
        # 定义一个简单的翻转函数：把 nums 从 start 到 end 的部分翻转
        def reverse(start, end):
            while start < end:
                nums[start], nums[end] = nums[end], nums[start]
                start += 1
                end -= 1
        
        # 2. 魔法三步走
        # 第一步：整体翻转
        reverse(0, n - 1)
        
        # 第二步：翻转前 k 个 (索引 0 到 k-1)
        reverse(0, k - 1)
        
        # 第三步：翻转剩下的 (索引 k 到 n-1)
        reverse(k, n - 1)
```

```python
class Solution:
```
```python
    def productExceptSelf(self, nums: List[int]) -> List[int]:
        n = len(nums)
        # 1. 初始化答案数组，先把所有位置填上 1
        answer = [1] * n
        
        # --- 第一轮：从左往右扫 ---
        # 计算每个位置【左边】所有数的乘积
        # answer[i] 应该等于 nums[0] * ... * nums[i-1]
        for i in range(1, n):
            # 现在的 answer[i] = 左边的 answer[i-1] * 左边的 nums[i-1]
            answer[i] = answer[i - 1] * nums[i - 1]
        
        # 此时 answer 数组里已经是左侧乘积了：[1, 1, 2, 6]
        
        # --- 第二轮：从右往左扫 ---
        # R 用来记录【右边】所有数的乘积，刚开始右边没数，所以是 1
        R = 1
        
        # 倒着遍历：n-1, n-2, ... 0
        for i in range(n - 1, -1, -1):
            # 对于索引 i：
            # 最终结果 = (左边的积 answer[i]) * (右边的积 R)
            answer[i] = answer[i] * R
            
            # 更新 R：把当前的 nums[i] 乘进 R 里，给下一个位置用
            R = R * nums[i]
            
        return answer

```

# 17、缺失的第一个正数
![[Pasted image 20260203144147.png]]


```python
class Solution:
    def firstMissingPositive(self, nums: List[int]) -> int:
        n = len(nums)
        
        # --- 第一步：萝卜归位 ---
        for i in range(n):
            # 这是一个 while 循环，因为换回来的新数字可能还是需要换位置
            # 条件解释：
            # 1. 1 <= nums[i] <= n: 数字必须在有效范围内（负数和超级大的数直接忽略）
            # 2. nums[nums[i] - 1] != nums[i]: 目标位置上已经有正确的数字了吗？
            #    如果没有（比如两个3，第一个3归位了，第二个3就不用换了），才交换。
            #    这一步非常关键，防止死循环！
            while 1 <= nums[i] <= n and nums[nums[i] - 1] != nums[i]:
                # 目标索引
                target_index = nums[i] - 1
                # 交换 (把 nums[i] 放到它该去的地方)
                nums[i], nums[target_index] = nums[target_index], nums[i]
                
        # --- 第二步：查岗 ---
        for i in range(n):
            # 如果索引 i 上的数字不是 i + 1
            if nums[i] != i + 1:
                return i + 1
                
        # 如果所有人都对号入座了（比如 [1, 2, 3]），那缺失的就是下一个
        return n + 1
```


利用这个下标。让这个数字回到它应该在的地方去。
然后再去查，看看哪一个不匹配。不匹配的返回下标值加一。

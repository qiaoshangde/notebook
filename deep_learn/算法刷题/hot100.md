# 2、字母异位词分组
![[Pasted image 20260131144041.png]]


用到了字典哈希。
先遍历对所有单词进行排序，然后以排序后的词作为键，值是一个列表，然后判断字典中是否有这个键，然后将其添加到这个排序后的键对应的值（列表）中。这样所有的异位词组都会存到这个键对应的列表中。只需要取值，然后进行list就可以了。

代码
```python
class Solution:

    def groupAnagrams(self, strs: List[str]) -> List[List[str]]:
        mp = collections.defaultdict(list)
        for st in strs:

            key = "".join(sorted(st))

            mp[key].append(st)

        return list(mp.values())
```

在代码中用到了一个叫collections的库，这个库可以帮助我少点检查是否有这个键的操作。直接向里面添加，如果有就可以对这个键对应的值进行操作，如果没有，会新建一个这个键，直接进行操作。

defaultdict（list）这个括号中填写一个数据类型。一旦检测到没有这个键，新建的这个键对应的值的数据类型就是这个默认的数据类型。
soted 就是进行排序。不管扔给他的是字符串还是列表，其返回都是一个列表。




最快的排序的复杂度是O（nlogn）
hashset的查找速度是O（1）。直接执行以此查询就行。


什么是hashset。就是一个只有键，没有值得哈希表。（可以用来执行自动去重）
并且无序无索引。就像一个大麻袋，里面有很多数字，你只能问里面有没有这个数字。但是你不能问这个里面第一个的数字是多少。

hashmap字典，hashset是集合。

# 4、移动0
这是一个关于快慢指针的题目。

![[Pasted image 20260201134457.png]]



直觉解法：遇到0就给它移动到最后。（计算复杂度太高，o（n2））

```python
# 你的思路的代码实现（虽然能跑，但效率低）
class Solution:
    def moveZeroes(self, nums: List[int]) -> None:
        i = 0
        loop_count = len(nums)
        
        # 我们得手动控制循环次数，不然死循环
        processed = 0 
        while processed < loop_count:
            if nums[i] == 0:
                # 1. 把 0 拿出来 (这一步很慢！O(N))
                nums.pop(i)
                # 2. 加到最后
                nums.append(0)
                # 3. 注意！因为删了一个元素，后面的补上来了
                # 所以 i 不需要 +1，我们要检查补上来的这个新元素
            else:
                # 只有不是 0 的时候，指针才往后走
                i += 1
            
            processed += 1

```

双指针法

```python
class Solution:

    def moveZeroes(self, nums: List[int]) -> None:

        """

        Do not return anything, modify nums in-place instead.

        """

        # slow 指针：指向下一个存放“非零元素”的位置

        slow = 0

        # fast 指针：负责遍历数组寻找“非零元素”

        for fast in range(len(nums)):

            # 只有当找到非零元素时，才做事

            if nums[fast] != 0:

                # 把找到的这个非零元素，和 slow 位置的元素交换

                # 这样非零的就被挪前面了，0 就被换后面去了

                nums[slow], nums[fast] = nums[fast], nums[slow]

                # 既然这个位置坐好了人，slow 就往后移一位，准备迎接下一个

                slow += 1
```

这是一个关于快慢指针的题目。在这个题中有两个下标，一个下标只负责查看是不是0.如果不是0，就移动到前面（慢指针指向的位置，相当于快慢指针指向的数字交换位置）。如果是0就不动。

复杂度为O（n）。原地进行操作。
也不能移动O到最后，交换位置。这样会导致非0元素的原始顺序会发生变化。


# 5、盛水最多的容器
![[Pasted image 20260201140345.png]]


题目分析，从左右头开始扫描。必须淘汰矮的一边，因为只要矮的呢一边存在。不管另一边移动后下一个是多少，都没有用，因为这个矮的就是限制的最大瓶颈。

```python
class Solution:
    def maxArea(self, height: List[int]) -> int:
        # 1. 左右护法就位
        left = 0
        right = len(height) - 1
        
        max_water = 0
        
        # 2. 只要没撞上，就一直PK
        while left < right:
            # 计算当前宽度
            width = right - left
            
            # 计算当前水位（取决于短板）
            # 这一步通过比较，顺便决定了谁该被淘汰
            if height[left] < height[right]:
                current_height = height[left]
                # 左边矮，记录下当前面积，然后淘汰左边
                max_water = max(max_water, width * current_height)
                left += 1
            else:
                current_height = height[right]
                # 右边矮（或一样高），记录下当前面积，然后淘汰右边
                max_water = max(max_water, width * current_height)
                right -= 1
                
        return max_water
```

两个指针分别指向左右，然后进行移动，只需要对数据扫描一遍，就可以找到最多的盛水容器。



# 6、三数之和


![[Pasted image 20260201142238.png]]




这是一个双指针的题目
```python
class Solution:
    def threeSum(self, nums: List[int]) -> List[List[int]]:
        # 1. 如果还没开始这就少于3个人，直接解散
        if not nums or len(nums) < 3:
            return []
        
        # 2. 关键第一步：排序！
        # 只有排序了，才能用双指针，才能方便去重
        nums.sort()
        
        result = []
        n = len(nums)
        
        # 3. 遍历每一个“老大”
        for i in range(n):
            # 如果老大自己都大于0，那后面肯定都大于0（因为排过序了）
            # 三个大于0的数加起来不可能等于0，直接结束
            if nums[i] > 0:
                break
                
            # 4. 老大去重：如果这个数和上一个数一样，跳过
            # i > 0 是为了保证 i-1 有效
            if i > 0 and nums[i] == nums[i-1]:
                continue
                
            # 5. 定义左右护法（双指针）
            # L 在老大右边第一个，R 在最后
            L = i + 1
            R = n - 1
            
            while L < R:
                total = nums[i] + nums[L] + nums[R]
                
                if total == 0:
                    # 找到了一组！
                    result.append([nums[i], nums[L], nums[R]])
                    
                    # 6. 护法去重（关键细节！）
                    # 比如 [-1, 0, 0, 0, 1]，找到一个0后，后面两个0都要跳过
                    while L < R and nums[L] == nums[L+1]:
                        L += 1
                    while L < R and nums[R] == nums[R-1]:
                        R -= 1
                    
                    # 找到答案后，两人同时收缩，找下一组
                    L += 1
                    R -= 1
                    
                elif total < 0:
                    # 结果太小了，把左边的（小的）变大一点
                    L += 1
                else:
                    # 结果太大了，把右边的（大的）变小一点
                    R -= 1
                    
        return result
```


首先对这个列表进行排序，从小到大排序。
然后进行循环，如果这个数跟左右两个指针指向的数的和为0，那么就是找到了一组。
需要注意的是：左右指针相遇就是两个指针已经对整个列表进行了扫描，已经寻找完毕。
循环时，这个数和上一个数不能相同。相同的话会找到同样的三个数。（去重）
左右指针移动的时候也要去重。

```python
            return 0

        left = 0

        right = len(height) - 1

        # 2. 定义左右两边的“墙”的高度

        left_max = 0

        right_max = 0

        water = 0

        # 3. 往中间逼近

        while left < right:

            # 更新左右两边的最高墙

            left_max = max(left_max, height[left])

            right_max = max(right_max, height[right])

            # 4. 核心逻辑：谁矮算谁

            if left_max < right_max:

                # 左边墙矮，说明左边这个位置的水位由 left_max 决定

                # 累加水量：瓶颈高度 - 柱子本身高度

                water += left_max - height[left]

                left += 1

            else:

                # 右边墙矮，说明右边这个位置的水位由 right_max 决定

                water += right_max - height[right]

                right -= 1

        return water
```

# 8、无重复字符的最长子串
![[Pasted image 20260201164831.png]]

```python

class Solution:
    def lengthOfLongestSubstring(self, s: str) -> int:
        # 1. 也是用 Set 做“记账本”，查重速度快 O(1)
        window_chars = set()
        
        # 2. 定义左指针
        left = 0
        max_len = 0
        
        # 3. 右指针主动去探索 (0 到 结束)
        for right in range(len(s)):
            current_char = s[right]
            
            # 4. 如果发现新来的字符已经在窗口里了（撞车了！）
            # 就要不断缩小左边，直到把那个重复的字符“挤出去”
            while current_char in window_chars:
                # 移除最左边的字符
                window_chars.remove(s[left])
                # 左指针向右缩
                left += 1
            
            # 5. 现在安全了，把新字符加进来
            window_chars.add(current_char)
            
            # 6. 量一下现在的宽度，挑战一下历史纪录
            # 宽度计算公式：right - left + 1
            max_len = max(max_len, right - left + 1)
            
        return max_len
```


在这个题目中，维护了两个指针，两个指针合在一起，相当于是一个滑动的窗口，一开始只移动右窗口，添加新的元素。一旦发现添加的元素中有已经含有的元素。则不断移动左窗口，知道把重复的旧元素排除出去。这个过程中，时刻计算窗口的长度，右指针减去左指针加一，并于已知的最大窗口进行比较，遍历一遍，即可得到最大的窗口。



# 9、找到字符串中的所有字母异位词
![[Pasted image 20260201165945.png]]



这个题目还是滑动窗口的题目。

```python
class Solution:
    def findAnagrams(self, s: str, p: str) -> List[int]:
        n, m = len(s), len(p)
        
        # 1. 边界条件：如果 s 比 p 还短，那肯定没有，直接回空
        if n < m:
            return []
        
        # 2. 准备两个 DNA 记录表 (长度26的数组，初始都是0)
        p_count = [0] * 26
        window_count = [0] * 26
        
        # 3. 先把 p 的 DNA 录入，同时把 s 的前 m 个字符（第一个窗口）录入
        for i in range(m):
            p_count[ord(p[i]) - ord('a')] += 1
            window_count[ord(s[i]) - ord('a')] += 1
            
        res = []
        
        # 4. 检查第一个窗口是否匹配
        if p_count == window_count:
            res.append(0)
            
        # 5. 开始滑动！从第 m 个字符开始，一直滑到最后
        # i 代表当前窗口【新加入】的那个字符的索引
        for i in range(m, n):
            # A. 进货：把新字符加入窗口
            new_char = s[i]
            window_count[ord(new_char) - ord('a')] += 1
            
            # B. 出货：把最左边被抛弃的字符移出窗口
            # 最左边的字符索引是 i - m
            old_char = s[i - m]
            window_count[ord(old_char) - ord('a')] -= 1
            
            # C. 比对：看看现在的窗口和 p 是否一样
            # 在 Python 里，比较两个列表的内容是否相同，直接用 == 就行，效率很高
            if window_count == p_count:
                # 记录起始索引 (就是 i - m + 1)
                res.append(i - m + 1)
                
        return res

```

在这个题目中，就是制作了一个和字串一样大小的窗口。然后制作了一个长为26的列表，然后默认第一位置的值就是a出现的次数。第二个位置的值就是b出现的次数，
 两个列表完全一样的时候，就说明这个符合添加，字串找到。记录开始的索引。 
 
 通过ord（）函数，就可以求出对应字符的ascll码。


# 10、和为K的子数组
![[Pasted image 20260201171413.png]]



看到题的第一反应是使用滑动窗口来做。但是不可以，因为有负的值。

一旦遇到负值，滑动窗口的指针就不知道移动哪一个。


```python
from collections import defaultdict

class Solution:
    def subarraySum(self, nums: List[int], k: int) -> int:
        # 1. 记账本：记录 {前缀和 : 出现的次数}
        # 核心细节：必须预设 {0: 1}
        # 含义是：前缀和为 0 的情况已经出现过 1 次了（就是刚开始还没动的时候）
        prefix_map = defaultdict(int)
        prefix_map[0] = 1
        
        current_sum = 0
        count = 0
        
        # 2. 遍历数组
        for num in nums:
            # 计算当前走到这里的累加和
            current_sum += num
            
            # 3. 核心公式查询：
            # 想知道有没有一个子数组和是 k，
            # 就看以前有没有出现过前缀和等于 (current_sum - k)
            target = current_sum - k
            
            # 如果以前出现过 target，说明中间这一段的和刚好就是 k
            if target in prefix_map:
                count += prefix_map[target]
            
            # 4. 把当前的前缀和记录到账本里
            prefix_map[current_sum] += 1
            
        return count

```


在这个题目中，建立了一个字典，key为前缀和，V为这个前缀和出现了几次。（可以观察到，如果值全都是正值，那么前缀和就是单调的，在这个字典中所有的值对应的都是1.）正是因为有负值的存在，所以导致前缀和的value不全为1.

一直求前缀和，找出当前前缀和减去之前某个位置的前缀和，就是这个子数组的前缀和。
因此，就可以转化为由当前前缀和减去k，就是某个要找的前缀和。如果字典中有，就说明这个前缀和出现过，就说明，字串存在。


# 11、滑动窗口最大值
![[Pasted image 20260202131218.png]]




```python
from collections import deque

class Solution:
    def maxSlidingWindow(self, nums: List[int], k: int) -> List[int]:
        # 这里的 q 就是那个“精英名单”，里面存的是【下标】！
        # 为什么要存下标？因为我们要通过下标算它是不是过期了。
        q = deque() 
        result = []
        
        for i in range(len(nums)):
            # 1. 【踢弱者】
            # 只要队列里有人，而且队尾的那个人比新来的 nums[i] 小
            # 那队尾那个人就永远没机会出头了，直接淘汰
            while q and nums[q[-1]] < nums[i]:
                q.pop() 
            
            # 2. 【进新人】
            # 现在的 nums[i] 有资格排队了
            q.append(i)
            
            # 3. 【清过气】
            # 队头是最老的。如果队头的下标 等于 i - k
            # 说明它刚好滑出了窗口的左边界，必须退休
            # 举例：k=3，当前 i=3 (第4个数)，那 i-k=0 (第0个数) 必须走了
            if q[0] == i - k:
                q.popleft()
            
            # 4. 【记录结果】
            # 从第 k-1 个元素（也就是窗口刚满）开始，每次都记下队头
            if i >= k - 1:
                result.append(nums[q[0]])
                
        return result
```


在这个题目中，考到了滑动窗口的知识点。
一个个加入到窗口。如何加入？，如果这个值比窗口内的所有值都大。就说明窗口内的值不起作用了。就把这个放到队头。其余的全部失效。如果不比第一个值大，但是比其余的值大，在窗口内时，也可以加入，但是也要按照大小顺序。（因为第一个有可能不被挤出但是时效到了，被踢出来了，这个时候，排名第二的数字，就可以顺利继位，变成第一个）
但是这个值当队头是有时效性的。如果在不被后加的值踢出的情况下。其最多可以呆一个滑动窗口的大小。（也就是新来的值的下标减去窗口的大小为最大值得下标的时候，意味着它已经呆了一个滑动窗口的大小了，马上要出窗口了）

进队前铲除异己（比我弱的全踢），进队后听天由命（等着前面的大佬退休）。

知识补充：在进队的时候，会把自己小的值全部踢出。因为这个时候，比自己小的值，再也不会有当队头的机会了，所以会直接踢出。
可以理解为，这个队列 就是一个最大值，以及最大值候选队列。每个一进来，就跟最后一个比，比它小的踢出，逐个和前面的比，直到找到自己的位置（其后面的全部踢出）。然后等待队列前面的大佬退休。自己继位。

```python
            # --- 进货环节 ---

            c = s[right]

            right += 1

            # 如果进的货是欠条里需要的，就记账

            if c in need:

                window[c] += 1

                # 如果这个字符的数量刚好凑够了，valid + 1

                if window[c] == need[c]:

                    valid += 1

            # --- 断舍离环节 ---

            # 当所有债务都还清了 (valid == len(need))，尝试缩小窗口

            while valid == len(need):

                # 1. 更新最小长度

                if right - left < min_len:

                    start = left

                    min_len = right - left

                # 2. 准备扔掉左边的货

                d = s[left]

                left += 1

                # 如果扔掉的货是欠条里的

                if d in need:

                    # 如果扔之前刚好够，扔了就不够了，valid - 1

                    if window[d] == need[d]:

                        valid -= 1

                    window[d] -= 1

        return "" if min_len == float('inf') else s[start : start + min_len]
```


这个题目，仍然是滑动窗口的题目

这个题目维护了两个值，一个是当前窗口内的字母的分布。（每个字母出现的字数）。
一个是（valid 表示：已经有几种字符的数量达标了？，比如t为AAB，这个时候need={‘A’：2，‘B’：1}，如果valid等于2，就表示A有两个B有一个）。

不断地将字母加入到窗口中，并统计符合条件的字母个数。合格时，就记录当前的下表和长度。然后将前面的出窗口。这里分为三种情况。一种是不是需要的字母，直接出。一种是是需要的字母，但是其出去后，窗口内的字母仍满足条件（这个字母本来就有富裕，出去一个没问题）。还有一种是是需要的字母，并且其一出去，条件就不满足，知道新的这个字母进来。

每次找到符合条件的情况，都记录下表和长度，直到找到最短的一版。（一旦新版比旧版短，就更新）。

```python
class Solution:
    def maxSubArray(self, nums: List[int]) -> int:
        # 1. 初始化
        # pre_sum 表示：必须包含当前这个数字的话，前面的最大和是多少
        # 刚开始就是第一个数
        pre_sum = nums[0]
        
        # max_sum 记录历史出现过的巅峰值
        max_sum = nums[0]
        
        # 2. 从第二个数开始遍历
        for i in range(1, len(nums)):
            # --- 核心决策 ---
            # 如果之前的累加和大于0，说明是“良性资产”，我们就加上它
            if pre_sum > 0:
                pre_sum = pre_sum + nums[i]
            # 如果之前的累加和小于等于0，说明是“不良资产”，只会拖累我
            # 那就不带它玩了，从当前数字 nums[i] 另起炉灶
            else:
                pre_sum = nums[i]
            
            # --- 记录巅峰 ---
            # 每次走一步，都要看看现在的身价是不是历史最高的
            max_sum = max(max_sum, pre_sum)
            
        return max_sum
```

```python
class Solution:
    def merge(self, intervals: List[List[int]]) -> List[List[int]]:
        # 1. 没有任何区间，直接返回空
        if not intervals:
            return []
        
        # 2. 关键一步：排序！
        # 按照每个区间的【左端点】从小到大排序
        # x[0] 表示取每个子列表的第0个元素（开始时间）
        intervals.sort(key=lambda x: x[0])
        
        merged = []
        
        # 3. 遍历每一个区间
        for interval in intervals:
            # 如果列表为空，或者当前区间的开始时间 > 上一个区间的结束时间
            # 说明【没有重叠】，直接把当前区间加进去
            if not merged or interval[0] > merged[-1][1]:
                merged.append(interval)
            else:
                # 否则，说明【有重叠】
                # 我们不需要添加新区间，而是修改上一个区间的结束时间
                # 谁结束得晚，就听谁的（取最大值）
                merged[-1][1] = max(merged[-1][1], interval[1])
        
        return merged
```


人的常理，像时间即可。先对这个按开始时间进行排序，然后一个个加入合并后的区间，一旦合并后的区间最后一个和新的区间有重叠就取最大的一个。

```python
        # 第三步：翻转剩下的 (索引 k 到 n-1)
```

# 16、除了自身以外数组的乘积
![[Pasted image 20260203143243.png]]

```python

class Solution:
    def productExceptSelf(self, nums: List[int]) -> List[int]:
        n = len(nums)
        # 1. 初始化答案数组，先把所有位置填上 1
        answer = [1] * n
        
        # --- 第一轮：从左往右扫 ---
        # 计算每个位置【左边】所有数的乘积
        # answer[i] 应该等于 nums[0] * ... * nums[i-1]
        for i in range(1, n):
            # 现在的 answer[i] = 左边的 answer[i-1] * 左边的 nums[i-1]
            answer[i] = answer[i - 1] * nums[i - 1]
        
        # 此时 answer 数组里已经是左侧乘积了：[1, 1, 2, 6]
        
        # --- 第二轮：从右往左扫 ---
        # R 用来记录【右边】所有数的乘积，刚开始右边没数，所以是 1
        R = 1
        
        # 倒着遍历：n-1, n-2, ... 0
        for i in range(n - 1, -1, -1):
            # 对于索引 i：
            # 最终结果 = (左边的积 answer[i]) * (右边的积 R)
            answer[i] = answer[i] * R
            
            # 更新 R：把当前的 nums[i] 乘进 R 里，给下一个位置用
            R = R * nums[i]
            
        return answer
```


先算出左边值的乘积再算出右边的值的乘积。然后左右相乘，就是所有值的乘积 



# 17、缺失的第一个正数
![[Pasted image 20260203144147.png]]


```python
class Solution:
    def firstMissingPositive(self, nums: List[int]) -> int:
        n = len(nums)
        
        # --- 第一步：萝卜归位 ---
        for i in range(n):
            # 这是一个 while 循环，因为换回来的新数字可能还是需要换位置
            # 条件解释：
            # 1. 1 <= nums[i] <= n: 数字必须在有效范围内（负数和超级大的数直接忽略）
            # 2. nums[nums[i] - 1] != nums[i]: 目标位置上已经有正确的数字了吗？
            #    如果没有（比如两个3，第一个3归位了，第二个3就不用换了），才交换。
            #    这一步非常关键，防止死循环！
            while 1 <= nums[i] <= n and nums[nums[i] - 1] != nums[i]:
                # 目标索引
                target_index = nums[i] - 1
                # 交换 (把 nums[i] 放到它该去的地方)
                nums[i], nums[target_index] = nums[target_index], nums[i]
                
        # --- 第二步：查岗 ---
        for i in range(n):
            # 如果索引 i 上的数字不是 i + 1
            if nums[i] != i + 1:
                return i + 1
                
        # 如果所有人都对号入座了（比如 [1, 2, 3]），那缺失的就是下一个
        return n + 1
```


利用这个下标。让这个数字回到它应该在的地方去。
然后再去查，看看哪一个不匹配。不匹配的返回下标值加一。

```python
            return []

        # 2. 只要墙没撞上，就一直转

            # 走完这一行，上边界下移

            top += 1

            # B. 向下走 (右边界)

            for i in range(top, bottom + 1):

                res.append(matrix[i][right])

            # 走完这一列，右边界左移

            # --- 关键检查 ---

            if top > bottom or left > right:

                break

            # C. 向左走 (下边界)

            for i in range(right, left - 1, -1):

                res.append(matrix[bottom][i])

            # 走完这一行，下边界上移

            bottom -= 1

            # D. 向上走 (左边界)

            for i in range(bottom, top - 1, -1):

                res.append(matrix[i][left])

            # 走完这一列，左边界右移

            left += 1

        return res
```

# 22、相交链表

![[Pasted image 20260205131956.png]]



```python
class Solution:
    def getIntersectionNode(self, headA: ListNode, headB: ListNode) -> ListNode:
        # 1. 边界检查：如果有一个为空，肯定不相交
        if not headA or not headB:
            return None
        
        # 2. 定义两个指针
        pA, pB = headA, headB
        
        # 3. 只要没相遇，就一直走
        # 即使不相交，最后 pA 和 pB 都会同时变成 None，循环也会结束 (None == None)
        while pA != pB:
            # pA 走法：如果走到头了(None)，就跳到 headB；否则往前走一步
            pA = headB if pA is None else pA.next
            
            # pB 走法：如果走到头了(None)，就跳到 headA；否则往前走一步
            pB = headA if pB is None else pB.next
            
        # 4. 相遇点就是交点（或者都是 None）
        return pA
```

- **指针 A 的路程**：先走 A (`a+c`)，再走 B (`b`) $\rightarrow$ 总共走到交点也就是 `a + c + b`。
    
- **指针 B 的路程**：先走 B (`b+c`)，再走 A (`a`) $\rightarrow$ 总共走到交点也就是 `b + c + a`。
    

**你看！ `a + c + b` 等于 `b + c + a`！**

只要它俩这样走，就一定会**在交点处同时相遇**。

```python
class Solution:
    def reverseList(self, head: Optional[ListNode]) -> Optional[ListNode]:
        # 1. 准备两个指针
        prev = None
        curr = head
        
        while curr:
            # 2. 关键步骤：先存下后面的路，防止断链
            temp = curr.next
            
            # 3. 修改指向：让当前节点指回前一个
            curr.next = prev
            
            # 4. 指针整体后移
            prev = curr
            curr = temp
            
        # 5. 最后 prev 会停在新的头节点上
        return prev
```


维护了两个指针，一个是当前指针，一个是当前指针的前一个指针。首先用temp记录当前节点的下一个节点，（第一步、因为要修改当前节点的下一个节点，先保存记录下来，直接修改就断链了）。然后让当前指针的下一个节点指向前一个节点。然后prev和curr一起后移动一个。

```python
class Solution:

        # 如果链表为空，或者只有一个节点，那反转后还是它自己
        if not head or not head.next:

        # 4. 断开原来的连接，防止形成环 (1->2->1)
        head.next = None
        
        return new_head
```

```python
class Solution:
    def isPalindrome(self, head: Optional[ListNode]) -> bool:
        if not head or not head.next:
            return True
            
        # 1. 快慢指针找中点
        slow, fast = head, head
        while fast and fast.next:
            slow = slow.next
            fast = fast.next.next
            
        # 此时 slow 就是后半段的起点
        
        # 2. 反转后半段链表 (直接复用反转链表的逻辑)
        # 我们定义一个 helper 或者直接写在里面都可以
        prev = None
        curr = slow
        while curr:
            temp = curr.next
            curr.next = prev
            prev = curr
            curr = temp
            
        # 反转结束后，prev 就是后半段的新头（原来的尾巴）
        
        # 3. 前后半段逐个比对
        left, right = head, prev
        while right: # 只要右边没走完（因为右边通常比左边短或相等）
            if left.val != right.val:
                return False
            left = left.next
            right = right.next
            
        return True
```

使用快慢指针找中点。然后使用迭代法（上一题），将后半部分翻转。最后逐个比对。

# 25、环形链表
![[Pasted image 20260205135650.png]]






```python
class Solution:
    def hasCycle(self, head: Optional[ListNode]) -> bool:
        # 1. 边界条件：如果没人或者只有一个人，肯定成不了环
        if not head or not head.next:
            return False
            
        # 2. 定义快慢指针
        slow = head
        fast = head
        
        # 3. 只要快指针还能跑（没到终点），就一直跑
        # 为什么是 fast.next？因为 fast 一次跑两步，所以要保证下下步也能走
        while fast and fast.next:
            slow = slow.next      # 慢走一步
            fast = fast.next.next # 快走两步
            
            # 4. 检查是否相遇（套圈了）
            if slow == fast:
                return True
                
        # 5. 如果循环结束了，说明 fast 冲过了终点（遇到 None）
        # 那就是没有环
        return False
```

只要有环，一个一次走两步，一个一次走一步。终有一时会套圈。

# 26、环形链表II
![[Pasted image 20260205140446.png]]


```python
class Solution:
    def detectCycle(self, head: Optional[ListNode]) -> Optional[ListNode]:
        slow, fast = head, head
        
        # --- 第一阶段：判断是否有环 ---
        while fast and fast.next:
            slow = slow.next
            fast = fast.next.next
            
            # 如果相遇了，说明有环
            if slow == fast:
                
                # --- 第二阶段：找入口 ---
                # 1. 让其中一个指针回到起点
                p = head
                # 2. 另一个指针留在相遇点
                q = slow
                
                # 3. 两个人都一步一步走
                while p != q:
                    p = p.next
                    q = q.next
                
                # 4. 再次相遇点就是入口
                return p
        
        # 如果跑出来了，说明没环
        return None
```


假设：

- **L**：从头节点到环入口的距离（我们要求的）。
    
- **C**：环的长度。
    
- **X**：从环入口到相遇点 M 的距离。
    

**当它们相遇时：**

- **慢指针 (`slow`) 走了**：$L + X$
    
- **快指针 (`fast`) 走了**：$L + X + n \cdot C$ （它多跑了 $n$ 圈）
    

因为快指针速度是慢指针的 2 倍，所以路程也是 2 倍：

$$2 \times (L + X) = L + X + n \cdot C$$

化简一下：

$$L + X = n \cdot C$$

$$L = n \cdot C - X$$

**这就很有意思了：**

$$L = (n-1) \cdot C + (C - X)$$

- $L$ 是头到入口的距离。
    
- $C - X$ 是什么？是 **从相遇点 M 继续往前走回到入口的距离**。
    
- 这意味着：**“从头走到入口的距离”** 刚好等于 **“从相遇点走到入口的距离（可能多绕几圈）”**。
    

所以，一个从头走，一个从相遇点走，速度一样，必然会在入口撞见！

```python
class Solution:
    def mergeTwoLists(self, list1: Optional[ListNode], list2: Optional[ListNode]) -> Optional[ListNode]:
        # 1. 创建哨兵节点 (Dummy Head)
        # 它的值是多少不重要，因为它最后会被丢弃
        dummy = ListNode(-1)
        
        # curr 指针用来在生成的新链表上移动
        curr = dummy
        
        # 2. 只要两个链表都有值，就开始比对
        while list1 and list2:
            if list1.val <= list2.val:
                curr.next = list1      # 谁小就把谁接过来
                list1 = list1.next     # 被选中的那个指针往后移
            else:
                curr.next = list2
                list2 = list2.next
            
            # 新链表的指针也要往后移，准备接下一个
            curr = curr.next
            
        # 3. 收尾工作
        # 循环结束后，肯定有一个链表先空了，另一个可能还有剩
        # 因为剩下的肯定是有序的，直接接在后面就行
        curr.next = list1 if list1 else list2
        
        # 4. 返回哨兵节点的下一个，那才是真正的头
        return dummy.next
```

# 29、删除链表的倒数第N个结点


![[Pasted image 20260209125709.png]]




```python
# Definition for singly-linked list.
# class ListNode:
#     def __init__(self, val=0, next=None):
#         self.val = val
#         self.next = next

class Solution:
    def removeNthFromEnd(self, head: Optional[ListNode], n: int) -> Optional[ListNode]:
        # 1. 创建哨兵节点，指向 head
        dummy = ListNode(0, head)
        
        # 2. 初始化双指针，都指向 dummy
        slow = dummy
        fast = dummy
        
        # 3. 让 fast 先走 n + 1 步
        # 为什么要多走一步？因为我们要让 slow 停在被删除节点的【前一个】
        for _ in range(n + 1):
            fast = fast.next
            
        # 4. 两个指针同时移动，直到 fast 走到尽头 (None)
        while fast:
            slow = slow.next
            fast = fast.next
            
        # 5. 此时 slow 就在被删除节点的前一个位置
        # 执行删除操作
        slow.next = slow.next.next
        
        # 6. 返回哨兵节点的下一个（真正的头节点）
        return dummy.next

```
这个使用了快慢指针。正常来讲我们只知道链表的头。删除倒数第N个。我们可能需要知道这个链表的长度。
使用快慢指针，可以先让快指针走N+1步，然后两个指针一起走。快指针指向空，说明慢指针到了删除结点的前一个结点。就可以将节点删除。

```python
            # 准备处理下一对 (3, 4)
```
